# מדריך למנכ״ל — מה אתה עושה, מה המערכת עושה

אין צורך בידע תכנות. עקוב אחרי השלבים לפי הסדר.

---

## שלב 0 — מה כבר נשמר

- **GitHub:** כל הקוד והשינויים ב-repo `vermillioncrm`
- **זיכרון AI:** `.cursor/rules/vermillion-crm-context.mdc` — כל סוכן Cursor יזכור את הארכיטקטורה והתוכנית
- **מסמך זה:** מה לעשות בכל פעם

---

## שלב 1 — וודא שיש מפתחות (פעם אחת)

פתח את הקובץ `.env` בתיקיית הפרויקט (Cursor כבר פתוח אצלך).

| משתנה | מה זה | מאיפה |
|--------|--------|--------|
| `VERMILLION_INGESTION_URL` | כתובת Supabase | Supabase → Settings → API → Project URL |
| `VERMILLION_INGESTION_SERVICE_KEY` | מפתח שרת (סודי!) | Supabase → Settings → API → `service_role` |
| `XAI_API_KEY` | מוח ה-AI (Grok) | חשבון xAI |
| `NEXTAUTH_SECRET` | סיסמת התחברות ל-CRM | כל מחרוזת ארוכה אקראית |

**מה אתה עושה:** אם חסר משהו — העתק מהדשבורד של Supabase / xAI. אל תשתף את המפתחות בוואטסאפ.

---

## שלב 2 — הפעלת ה-CRM (כל יום עבודה)

1. ב-Cursor: תפריט **Terminal** → **New Terminal**
2. הקלד (או העתק):

```powershell
cd c:\Users\97254\Desktop\vermillioncrm
npm run dev
```

3. פתח דפדפן: **http://localhost:3000**
4. התחבר כמנכ״ל

**מה אתה רואה בסרגל:** «מחובר כמנכ״ל», האם Supabase מחובר, האם יניקה פעילה.

---

## שלב 3 — סנכרון נתונים מהאפליקציה

1. לך ל-**משתמשי האפליקציה** (`/vermillion`)
2. לחץ **סנכרן עכשיו** (או כפתור הסנכרון בסרגל)

**מה קורה מאחורי הקלעים:** המערכת מושכת מ-Supabase פרופילים, Stamps, משחקים, ימים 9–30, ושומרת אצלך במחשב (SQLite). ה-AI קורא רק מהעותק המקומי.

**מתי ללחוץ:** אחרי שינוי גדול באפליקציה, או פעם ביום בבוקר.

---

## שלב 4 — בדיקה שהנתונים החדשים הגיעו

1. פתח משתמש אחד שיש לו ימים 9+ באפליקציה
2. בכרטיס המשתמש חפש:
   - **ימי תוכנית 9–30** (daily_logs)
   - **משחקים** עם שם משחק ו-«טוקן נוצל»
3. הרץ **סוכן VerMillion** בדשבורד — התשובה אמורה להיות מעמיקה יותר

---

## שלב 5 — מרכז AI

נתיב: **מרכז מנכ״ל → מרכז AI — תיעוד**

כאן רואים כל ריצת סוכן: שלבים, טוקנים, עלות משוערת.

## ידע מוצר — מה ה-CRM «יודע» על האפליקציה

נתיב: **מוצר VerMillion → ידע מוצר (חוקים)**

שם מופיעים: חוקי DNA, שישי/שבת, 31 משחקים, תקנון, anti-cheat.  
סוכן VerMillion משתמש בזה אוטומטית לפני כל ניתוח — לא צריך להסביר לו את המוצר בכל פעם.

---

## מה **לא** לעשות

- לא למחוק את `prisma/dev.db` (זה כל הנתונים המקומיים)
- לא לשתף `service_role` key
- לא לסגור את הטרמינל עם `npm run dev` בזמן שהמערכת מסנכרנת משתמשים רבים

---

## אם משהו נשבר

| תסמין | מה לעשות |
|--------|-----------|
| «יניקה: ממתין» | לחץ סנכרון; בדוק ש-`.env` מלא |
| שגיאה אחרי עדכון קוד | עצור dev (Ctrl+C), הרץ `npx prisma generate`, שוב `npm run dev` |
| משתמש לא מופיע | סנכרון + בדוק שהוא לא ב-«נטשו» |
